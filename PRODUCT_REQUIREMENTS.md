# ReadFocus 产品需求文档 (PRD)

## 1. 产品概述
**产品名称**：ReadFocus  
**核心理念**：将深度专注（Deep Work）与深度阅读（Deep Reading）结合，打造“专注即奖励”的正向反馈循环。  
**目标用户**：需要长时间专注工作/学习，且平时有使用微信读书习惯的用户。  

## 2. 核心功能
### 2.1 专注计时器 (Focus Timer)
- **多模式支持**：
    - **番茄钟 (Pomodoro)**：默认 25 分钟专注，适合标准工作流。
    - **自定义倒计时 (Custom)**：用户设定时长（如 45m, 60m），适合长任务。
    - **心流模式 (Focus Flow)**：正向计时，不设上限，记录由于心流产生的专注时长。
- **极简界面**：大字号时间显示，无干扰背景，提供开始/暂停/重置控制。

### 2.2 休息奖励 (Break & Reward)
- **触发机制**：专注计时结束（或心流模式手动停止）后，自动进入休息界面。
- **随机漫游卡片**：
    - **内容**：随机展示用户微信读书中的一条“划线”或“想法”。
    - **信息**：包含原文内容、书籍封面、书名、作者、划线时间。
    - **交互**：提供“下一张”按钮，允许用户在休息期间随机浏览，寻找灵感。

## 3. 技术架构
- **前端**：Next.js (React), Tailwind CSS, Lucide Icons。
- **后端**：Next.js API Routes (作为反向代理)。
- **数据源**：微信读书 API (WeChat Reading)。
- **鉴权**：通过 Cookie 本地存储鉴权 (`wr_skey` 等)。
- **隐私**：Cookie 仅存储在用户本地环境 (.env)，不上传至任何第三方服务器。

## 4. 关键流程
1. **配置**：用户获取微信读书 Cookie 并填入环境变量。
2. **专注**：选择模式 -> 开始计时 -> 完成任务。
3. **奖励**：系统随机抓取一本笔记本 -> 随机抓取一条划线 -> 渲染精美卡片。

## 5. 后续规划 (Roadmap)
- [ ] **数据可视化**：统计每日专注时长与阅读回顾数量。
- [ ] **书架视图**：允许用户指定“只回顾某几本书”。
- [ ] **PWA 支持**：支持安装到桌面/手机，离线也能使用基础计时功能。
